UPGRADE 1 — EXPERIENCE REPLAY Q-LEARNING

Implement an experience replay buffer to stabilize learning.

Create a ReplayBuffer class:

• capacity = 10,000 transitions
• store tuples: (state, action, reward, next_state, done)

When taxis act:

1. Push transition into buffer
2. Sample mini-batch (size 32) every few steps
3. Update Q-values using batch learning

Q-update formula:

Q(s,a) = Q(s,a) + α [ r + γ max Q(s′,a′) − Q(s,a) ]

Recommended hyperparameters:

learning_rate (α) = 0.1
discount (γ) = 0.95
batch_size = 32

Ensure replay buffer is memory efficient.
